Frequently Asked Questions

What is FusionScript?

FusionScript is a C++, Python, and Lua inspired language written to provide an alternative syntax to Lua. The language (currently) compiles to Lua source and eventually might compile to Lua VM instructions or use a modified Lua VM. The language itself was originally designed by Ryan, and written during a month-long trip to San Francisco. Here are some more points over the language:

  1. Dynamic Typing: The (currently unofficial) standard library includes an assertion library that provides a type checking method, assert.is(). It can be used to check the VM type as well as whether a constructed table is an instance of a specific class (or classes it inherits values from) or whether it implements methods specified by an interface.
  2. Classes and Interfaces: The language, when compiled to Lua source, uses a wrapper method to create class objects that can be called to create an instance. Classes can inherit values and methods from other classes and use an interface to enforce that values implement certain methods.
  3. Generators: Coroutines that generate values (also called a generator or iterator) can be made easily by appending the keyword async before the function name, and return values using the keyword yield. FusionScript also has support for creating array- and hashmap-like tables using the values from an iterable object ({x in y()} and {x = y for x, y in pairs(z)})

Why was FusionScript written?

The original author, Ryan, wanted a programming language that looked like C, had Python-like hashmap/array generation, and ran on the Lua VM. So, he wrote one. Also he wanted to be able to put it on his résumé.
